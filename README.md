# GrindOS

Геймифицированный гриндер для учёбы и дел. Вбиваешь тему → ИИ строит дерево
прогрессии → гриндишь короткими заходами и видишь рост на трёх циферблатах.
Реализация по `grindos-spec.md`.

## Запуск

```bash
npm install
cp .env.local.example .env.local   # вставь свой ANTHROPIC_API_KEY (опционально)
npm run dev                         # http://localhost:3000
```

Без ключа приложение работает: `/api/generate-tree` отдаёт офлайн-черновик
(6 узлов по теме). Ключ читается **только на сервере** в
`app/api/generate-tree/route.ts` и не попадает в клиентский бандл.

## Стек
- Next.js 14 (App Router) + TypeScript
- Tailwind CSS, lucide-react, recharts
- `@anthropic-ai/sdk` — только в серверном route handler
- Персист: `localStorage` (фолбэк в память), ключи `grindos:*`

## Структура
- `lib/engine.ts` — формулы XP, лут-ролл, доступность узлов, применение захода
- `lib/normalize.ts` — защищённый парс + нормализация ответа ИИ
- `lib/offline.ts` — офлайн-заглушка дерева (число узлов по сложности)
- `lib/store.ts` — localStorage-стор: гринды, индекс, глобальный профиль (вехи), lifetime XP
- `lib/useGrind.ts` — клиентский стейт-хук (заходы, детали, заметки, вехи)
- `lib/llm.ts` — общий серверный вызов провайдера (OpenRouter / Anthropic)
- `app/api/generate-tree/route.ts` — генерация дерева (сложность → структура и число квестов)
- `app/api/explain-node/route.ts` — ИИ-разбор отдельного квеста (кэшируется)
- `components/*` — HUD: шапка с уровнем, три циферблата, дерево, карта, модалки

## Страницы
- `/` — рабочий стол активного гринда (тиры дерева)
- `/map` — карта всего дерева (SVG-граф, прогресс, состояния узлов)
- `/projects` — история проектов: выбрать / удалить / создать
- `/rewards` — суммарный опыт, уровень аккаунта и награды-вехи (реальные призы)

## Смысл XP
Опыт со всех проектов копится в глобальный счёт аккаунта (уровень в шапке). На
страницы наград можно задать реальные призы на порогах XP — при достижении порога
во время захода выпадает праздничное «вскрытие награды».

## Темы интерфейса
Шестерёнка в шапке переключает HUD (тёмный slate) и **Pip-Boy** (зелёный CRT:
сканлайны, свечение, мерцание, моношрифт) с тумблером «спокойный режим». Палитра —
через CSS-переменные (`app/globals.css`), поэтому тема меняется без перерисовки
компонентов; выбор хранится в localStorage и применяется до первой отрисовки.

## Мини-тесты
В записи матча есть вкладка «Мини-тест»: `/api/generate-quiz` собирает квиз по
закрытым узлам, результат в % автоматически становится очком на кривой матчей.

## Редактор квестов
Кнопка «Редактировать» над деревом включает правку: карандаш на узле открывает
редактор (имя, тир, дрилл, unlock, XP, зависимости), есть «+ Добавить квест» и
удаление. Структура авто-валидируется (`sanitizeNodes` — циклы/висячие связи,
нумерация шагов, босс проекта).

## Генерация из плана
На стартовом экране режим «Готовый план»: вставь план по пунктам — ИИ разнесёт его
по узлам/тирам как проект (финал = босс), сохраняя порядок и формулировки.

## Квесты = пошаговый план
Узлы — это шаги реализации (декомпозиция темы/задачи), а не вечный гринд. ИИ помечает
флагом `repeatable` только те шаги, что по сути являются дриллами (отработка/заучивание);
в карточке у них бейдж «повтор», у остальных — «что сделать».

## Таймер фокуса
Заход открывает таймер обратного отсчёта (15/25/50) с паузой. XP начисляется за реально
отсиженные минуты: завершишь раньше — получишь частичный XP. Есть и кнопка «Засчитать
без таймера» (на доверии).

## Распад XP и баланс
XP делится на два счётчика: `earnedXP` (монотонный, для наград-вех) и `xpBalance`
(тратится/ржавеет, задаёт уровень аккаунта). После ~2 дней простоя незакрытые шаги и
баланс гуманно «ржавеют» каждый день (`lib/engine.ts → applyDecay`); при заходе показывается,
сколько просело.

## Таймер фокуса в Picture-in-Picture
Заход «Старт фокуса» запускает глобальный таймер (живёт в `lib/focus.tsx`, провайдер в
layout): он переживает переход между страницами, показывается плавающим виджетом и может
открепиться в окно **картинка-в-картинке** (Document Picture-in-Picture) — отсчёт виден
поверх других вкладок, пока работаешь дальше. Завершение начисляет XP через общий
`store.commitRun`; на странице гринда показывается лут, на других — короткий тост.

## Страница персонажа
Клик по чипу уровня (или вкладка «Персонаж») открывает `/character`: SVG-фигурку с бейджем
уровня, серию дней, «прокачанные статы» (SPECIAL-стиль: Фокус, Дисциплина, Эрудиция, Темп,
Меткость, Упорство — выводятся из активности), календарь-хитмап XP по дням за 12 недель,
бар-чарт последних 14 дней и сводку по всему (минуты, заходы, шаги, ставки). Логика —
`lib/stats.ts`.

## Ставки на дедлайн
В карточке шага можно поставить XP на «закрою к дедлайну» (1 день / 3 дня / неделя).
Успел → возвращается ставка +75%. Не успел → теряешь половину (гуманно). Ставка
удерживается из баланса; выигрыш засчитывается при закрытии шага, проигрыш — по истечении
дедлайна при загрузке.

## Заметка о безопасности
`npm audit` помечает весь мажор Next.js (advisory от 2025-12). Это DoS/SSRF-классы
для self-hosted прод-развёртываний; для локального одиночного MVP не критично.
Реальный фикс — апгрейд до Next 16 (ломающие изменения). Закреплён стабильный
`next@14.2.35`.
