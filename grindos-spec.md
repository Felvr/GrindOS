# GrindOS — спецификация системы

Геймифицированный гриндер для учёбы и дел. Вбиваешь любую тему или дело → система сама строит дерево прогрессии → ты гриндишь короткими заходами и **видишь, как растёт скилл**. Универсальная (тема не зашита заранее), с лутом и тремя циферблатами прогресса.

---

## 1. Главный принцип

Большинство геймификаций кринжовые, потому что клеят фейковые очки поверх учёбы: награда ни на что не влияет, мозг это просекает и забивает. Здесь два правила:

1. **Само действие = награда.** То, что ты гриндишь, и есть прокачка. Каждый заход делает следующий жирнее, открывает новые узлы.
2. **Прогресс должен быть виден сразу.** В играх ты видишь конвертацию гринда в силу за минуты; в жизни результат приходит через месяцы и невидим — отсюда теряется концентрация. Задача системы — сделать невидимое видимым и сократить петлю обратной связи до секунд/дней.

Три эталонные петли (на них завязана вся механика прогресса):

- **GTA** (ограбления → катаешься на тачке): гринд конвертируется в *конкретную способность, которой ты пользуешься*. → за каждый закрытый узел выдаём **реальное «теперь умею…»**, а не абстрактный левел.
- **CS** (aim_botz → выносишь в матче): скилл из дрилла переносится в «настоящий матч», результат виден. → разделяем **дриллы** и **матчи**, рисуем кривую результатов матчей.
- **Шахматы** (насмотренность): прогресс = объём увиденного. → отдельный видимый **счётчик повторений/объёма**.

---

## 2. Универсальность: skill vs project

При создании гринда ИИ классифицирует ввод:

- **skill** — навык/знание («испанский с нуля», «теория графов», «бэкенд на Go»). Строится **дерево зависимостей**: узлы разных тиров, поздние требуют ранние.
- **project** — конкретное дело с дедлайном («сдать диплом к пятнице», «собрать портфолио»). Строится **линейная цепочка квестов**, финальный узел = **БОСС** (сам дедлайн/сдача).

Разница только в форме графа и подписях (в project «Гараж» называется «Сделано», последний узел помечается боссом). Остальные механики общие.

---

## 3. Три циферблата (ядро видимого прогресса)

Панель сверху, всегда на виду. Три разных таймскейла фидбека.

| Циферблат | Что показывает | Петля из |
|---|---|---|
| **Гараж** | Карточки закрытых узлов: имя + `unlock` («теперь могу заказать на испанском»). Растёт по мере закрытия. | GTA |
| **Кривая матчей** | Линейный график результатов «матчей» (0–100) по порядку. Растущая линия = доказательство роста. | CS |
| **Насмотренность** | Большой счётчик минут в фокусе + число заходов. Тупо объём, сделанный видимым. | Шахматы |

Плюс под счётчиком — мини-полоска последних дропов (цветные квадратики по редкости) для ощущения накопления.

## 4. Петли (loops)

### Заход (run)
Сессия фокуса. Открывается кликом по доступному узлу. Параметры:
- длина: **15 / 25 / 50 мин** (патруль / данж / рейд);
- честный тоггл **«телефон убран?»** (yes/no) — анти-отвлечение, без слежки, на доверии.

### Анти-отвлечение (combo)
Телефон убран → **комбо ×2** к XP и сдвиг шансов лута в сторону редкого. Телефон рядом → ×1, базовый лут. Это превращает удержание фокуса в ресурс, который жалко терять.

### Левелинг
```
base = round(minutes * (phoneFree ? 2 : 1) * 1.2)
loot = rollLoot(base, phoneFree)
gain = base + loot.bonus
progress[node] = min(node.xp, progress[node] + gain)
```
Когда `progress[node] >= node.xp` → узел **закрыт**: попадает в Гараж (имя + unlock), разблокирует зависимые узлы.

### Лут-ролл (вариативность вместо рилсов)
После каждого захода — ролл дропа. Это дофаминовый спайк, конкурирующий со свайпами.
```
r = random()                       // [0,1)
if phoneFree: r = r*0.6 + 0.4      // скос к редкому
r > 0.997 → ЛЕГЕНДА   (bonus mult 2.2)
r > 0.95  → Эпик       (1.1)
r > 0.82  → Редкий     (0.6)
r > 0.55  → Необычный  (0.3)
else      → Хлам       (0.0)
bonus = round(base * mult)
```
Показывать ролл отдельным «вскрытием сундука» (цвет по редкости, +XP, +бонус).

### Доступность узлов
`available` ⇔ все `requires` уже в `cleared`. Иначе заблокирован, показываем «Сначала: <имена>».

### Матч (проверка в бою)
Отдельная кнопка «+ матч». Пользователь применяет навык по-настоящему и ставит себе **0–100**. Запись уходит в кривую. Система подсказывает форматы матчей из сгенерированного `matches[]`.

### Престиж (длинная петля)
Когда закрыты все узлы (или повержен босс) — гринд архивируется, выдаётся **перманентный множитель** (мета-скилл: быстрее учишься, лучше держишь фокус). Стартуешь новую тему с бонусом. Фрейм «прокачка между забегами».

## 5. Генерация дерева (AI)

Один вызов LLM на создание гринда. Ответ — **минифицированный JSON**, парсить защищённо (срезать ```` ```json ````, взять от первой `{` до последней `}`).

### Схема ответа
```json
{
  "type": "skill | project",
  "title": "string",
  "summary": "string",
  "nodes": [
    {
      "id": "string",
      "name": "<=6 слов",
      "tier": 1,                 // 1..4, tier1.requires = []
      "requires": ["id"],
      "grind": "<=14 слов — что КОНКРЕТНО повторять, как дрилл",
      "xp": 100,                 // 60..200
      "unlock": "<=14 слов — что реально СМОЖЕШЬ делать после"
    }
  ],
  "matches": [
    { "atTier": 1, "name": "<=6 слов", "how": "<=16 слов — как проверить себя в реальном применении" }
  ]
}
```

### Промпт генератора (суть)
> Ты — дизайнер игровой прогрессии. По теме/делу верни ТОЛЬКО минифицированный JSON по схеме выше. Определи `type`: skill (дерево зависимостей) или project (цепочка квестов, последний узел — босс). 6–7 узлов. Весь текст на русском, кратко. tier 1..4, узлы tier 1 без requires, зависимости логичны. `grind` — конкретный повторяемый дрилл. `unlock` — конкретная способность, не абстракция. matches: 2–3 шт. Только JSON, без markdown и пояснений.

Ограничения: держать `max_tokens ≈ 1000`, поэтому ≤7 узлов и короткие поля (иначе JSON обрежется). Нормализовать ответ (клампить tier/xp, дефолты на пустые поля). Если вызов недоступен — офлайн-заглушка из 6 узлов по теме, помеченная как черновик.

---

## 6. Модель данных

**Grind** (один на тему):
- `slug`, `title`, `type`, `summary`, `createdAt`
- `tree`: `{ nodes[], matches[] }` (из генератора)
- `progress`: `{ [nodeId]: xpCurrent }`
- `cleared`: `[nodeId]`
- `garage`: `[{ name, unlock, t }]`
- `matches`: `[{ name, score, t }]`
- `reps`: int, `minutes`: int
- `lootLog`: `[{ rarity, t }]` (последние ~40)
- `prestigeMult`: number (после архивации)

**Index**: список `[{ slug, title, type }]` для переключения между гриндами.

## 7. Экраны / UI

Один экран, тёмный HUD-стиль (slate-фон, акценты: gold = лут/награда, cyan = прогресс, red = босс/опасность; моноширинный для цифр).

1. **Шапка**: лого + переключатель сохранённых гриндов.
2. **Поле ввода**: «Что качаем?» → кнопка «Сгенерировать».
3. **Три циферблата** (раздел 3).
4. **Дерево**: узлы сгруппированы по тирам (TIER 1..4). Карточка узла: иконка состояния (закрыт / доступен / заблокирован / босс), имя, `grind`-подсказка, XP-бар `xp/target`, кнопка «гриндить заход». Закрытый показывает зелёную `unlock`-строку.
5. **Модалки**: заход (длина + тоггл телефона), вскрытие лута, запись матча.
6. **Подсказки матчей** из `tree.matches`.
7. **Пустое состояние** и понятная ошибка генерации с кнопкой «собрать офлайн».

---

## 8. Тех-требования (агностично к стеку)

- **Персист** по гринду: key-value, async ок. Грузить индекс + последний гринд на старте, сохранять после каждого значимого действия. Фолбэк в память, если хранилище недоступно.
- **AI-вызов** возвращает JSON → парсить защищённо, нормализовать, обрабатывать ошибку/обрезку.
- **Графики** — любая лёгкая либа (кривая матчей).
- **Доступность**: адаптив до мобилы, видимый фокус, уважать `prefers-reduced-motion`. Анимации минимальные (вскрытие лута, всплытие анлока).

---

## 9. Не-цели и грабли

- **Логирование — в один тап.** Главная ловушка таких систем: трекер сам становится прокрастинацией. Заход = выбрать длину + тоггл + кнопка. Никаких форм на полэкрана.
- **Тоггл телефона — на доверии**, без слежки и без жёстких наказаний.
- **Стрики — опционально и мягко** (если делать: щиты/заморозка, чтобы пропуск дня не обнулял мотивацию). По умолчанию можно не делать.
- Не плодить валюты и меню ради меню — каждый элемент кодирует что-то настоящее (редкость = ценность, cyan-бар = заполнено).
